# /commit

변경된 파일을 분석해 기능 단위로 묶어 커밋한다.

## 커밋 메시지 형식

```
(feat|fix|refactor|docs|chore) 메인 코멘트 한 줄

- 세부 항목 1
- 세부 항목 2
```

- 타입은 변경 성격에 맞게 선택: `feat`(새 기능), `fix`(버그 수정), `refactor`(리팩터), `docs`(문서), `chore`(설정/기타)
- 메인 코멘트는 한국어, 한 줄
- 세부 항목은 `-`로 시작, 각 항목 한 줄 요약

## 워크플로우

### 1단계: 상태 파악
`git status`와 `git diff`로 변경 파일 및 내용 확인.

### 2단계: 기능 단위로 그룹화
변경 파일을 성격에 따라 묶는다. 서로 다른 기능/목적이면 커밋을 분리한다.

### 3단계: 커밋 실행
각 그룹에 대해 순서대로:
1. 해당 파일만 `git add`
2. 아래 형식으로 커밋

```bash
git commit -m "$(cat <<'EOF'
(feat) 메인 코멘트

- 세부 항목 1
- 세부 항목 2
EOF
)"
```

### 4단계: 푸시
`git push`로 리모트에 푸시한다. 리모트 브랜치가 없으면 `git push -u origin <브랜치명>`으로 설정한다.

### 5단계: 결과 보고
커밋 목록을 `git log --oneline`으로 확인 후 간단히 보고.
