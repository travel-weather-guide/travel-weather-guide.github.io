# /plan-status

PLAN-CURRENT.md의 현재 작업 진행도를 분석·보고한다.

## 사용법
```
/plan-status
```

## 워크플로우

### 1단계: 파일 로드
1. `PLAN-CURRENT.md`를 읽는다
2. 진행 중인 항목이 없으면 안내하고 종료

### 2단계: 체크리스트 분석

PLAN-CURRENT.md의 체크리스트를 파싱:
- `[x]` 완료 항목 수
- `[ ]` 미완료 항목 수
- 전체 대비 완료 비율 계산
- 하위 섹션별로 그룹화

### 3단계: 보고

아래 형식으로 사용자에게 보고한다:

```
## 현재 작업: {항목명}

> 시작: YYYY-MM-DD

| 섹션 | 완료 | 전체 | 상태 |
|------|------|------|------|
| 2-1. 작품 설명란 | 2 | 2 | ██████████ 완료 |
| 2-2. 패널 구조 A | 1 | 3 | ███░░░░░░░ 33% |

전체: **X/Y 완료 (Z%)**

### 남은 항목
- ⬜ 미완료 항목 1
- ⬜ 미완료 항목 2
```

진행률 바는 10칸 기준: █ = 완료, ░ = 미완료
