# Travel Weather - 구현 플랜

> 구체적 구현 계획은 `/start-phase`로 시작한 뒤 PLAN-CURRENT.md에서 진행.

---

- 목적지 비교 — 비교 항목 우선순위, 3개 초과 선택 시 UX
- 코멘트 강화 — 아이콘-날씨 매핑 규칙, 정보 과잉 방지
- 테마별 추천 — 테마 분류 기준, 한 지역이 여러 테마에 속할 때
- 데이터들 많아질경우 어떻게 관리할건지, 구조개편 가능한지
- 실제 사람들이 이용할만한, 경쟁력 가질만한지, 어떻게 해야할지
- SEO — 수백 페이지 메타데이터 자동화, 크롤링 우선순위
- 반응형 최적화 — 차트/테이블/지도 각각의 모바일 대응 전략
- 다국어 — 정적 데이터(JSON)의 다국어 구조, URL 전략
- 비자 정보 — 데이터 최신성 유지 방안, 면제/도착비자 구분
- 베스트 시즌 캘린더 — 여러 목적지를 한 뷰에 담을 때 정보 밀도
- 짐 싸기 체크리스트 — 기후→아이템 매핑 규칙, 개인 차이 반영
- 배포 — Cloudflare Pages + Next.js 호환성, 정적 export 제약
- PWA — 오프라인 캐시 범위, 데이터 업데이트 전략
- 항공권 최저가 + 날씨 매칭 — API 비용, 가격 데이터 신선도, 추천 로직
